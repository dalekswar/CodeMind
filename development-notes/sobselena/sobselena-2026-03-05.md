Сегодня занимался рефакторингом - заменил TanStack на RTK Query. Сначала немного терялся, потому что API у RTK Query отличается, но в итоге разобрался, как правильно его использовать: нужно создать API slice и настроить в нём запросы и мутации. Всё-таки по синтаксису он выглядит более громоздким по сравнению с TanStack Query 🙂

Сделал ещё одну приятную мелочь: после регистрации пользователь теперь сразу попадает на страницу login-page, а не на About Us. Теперь роутеры AuthorizedOnly и PublicOnly начинают работать только после авторизации на странице login-page, а не на любой из страниц авторизации.

После логина теперь получаем token вместо логина, который используем для всех последующих запросов к серверу.
